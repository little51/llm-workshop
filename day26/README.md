# 《基于大模型和智能体的客服项目》实训报告撰写要求

## 一、项目概述

### 1.1 项目背景
- 为什么选择用大模型和智能体将智能客户服务作为综合实训方向？当前智能客服的技术机遇是什么？
- 核心需求：人工客服成本高、服务的即时性不够、服务的规范性不足等

### 1.2 项目目标
构建一个什么场景的多模态智能服务系统？（如：电商平台智能客服、直播带货助手、跨境电商多语言导购、农产品电商助教服务等）

核心能力目标：**模型训练**、**客户服务知识库应用**、**语音/图文多模态交互**，辅以智能体路径决策、图像识别（OCR）、数字人等

预期达到的效果指标（如：问题解决率、用户满意度、语音交互成功率等）

### 1.3 团队分工

模型训练 、数据集处理、应用开发、数字人制作、声音克隆、应用测试、论文撰写、程序演示等

## 四、项目总结与反思

### 4.1 关键成果
- **技术成果**：成功训练了什么规模的模型？构建了多大的商品知识库？实现了哪些多模态能力（图文/语音）？
- **应用成果**：系统在什么场景下演示？用户/客户反馈如何？
- **学习成果**：通过本项目掌握了哪些核心技能？

### 4.2 经验总结
- **技术选型**：基础模型、训练策略、商品知识库方案的选择依据与反思
- **资源消耗**：各阶段耗时与算力成本分析（训练 / 知识库构建 / 系统集成 / 测试）
- **团队协作**：分工策略、遇到的最大挑战及解决思路
- **改进空间**：如果重新做一次，哪些地方会做得不一样？哪些技术选型会调整？
